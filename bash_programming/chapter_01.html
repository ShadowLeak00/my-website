üîÑ Streams and Redirection

Standard channels:

  • stdin: 0
  • stdout: 1
  • stderr: 2

Redirection examples:

command > file.txt       # stdout 
command >> file.txt      # append 
command 2> error.log     # stderr 
command > output.txt 2>&1  # redirect both 
sort < input.txt # redirect input.txt data to sort

By this we can send multiline data to a file. "EOF" keyword can be replaced as your choice

cat << EOF 
This is a test. 
Here document input. 
EOF

  • & # run in background
  • && # run if previous succeeded
  • || # run if previous failed
  • ; # command separator
  • ;; # end of case option
  • | # pipe output to next command

"()" use to group few commands and ";" use to seperate each command

(ls; cd mydir; cat text.txt) | grep "word"

üåê Sample Script: Ping Target

#!/bin/bash
target="${1}"
scriptName="${0}"
        
if [[ -n "$scriptName" && -n "$target" ]]; then
  echo "Running ${scriptName}....."
  echo "Pinging ${target}.........."
  ping "$target"
else
  cat << EOF
  Usage: ./${scriptName} 192.168.1.1
  EOF
fi

üî¢ Special Variables

  • $0 - script name

  • $#- number of arguments
  • $* - all arguments as one word
  • $@ - all arguments individually quoted

  echo "${args}"
